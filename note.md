# Role
Act as a Senior Product Manager and Lead Developer specializing in ADHD Neurodivergent Productivity Tools.

# Source Material Context
Based on the "Energy Sprints & Kaizen System" methodology:
- Rejects "Time Blocking" (rigid schedules fail due to time blindness/variable energy).
- Uses "Energy Buckets" instead of task types.
- Core philosophy: "Follow your energy, but respect the guard rails."

# Project Goal
Build a Single Page Application (SPA) called "Kaizen Flow" that manages tasks based on emotional energy and urgency, not calendar slots.

# Functional Requirements (Must-Have)

1.  **Brain Dump Zone (The Inbox)**
    - Input: Fast text entry (reduce working memory load).
    - Default State: All new items go to "Unsorted".

2.  **The 4 Energy Buckets (Sorting Logic)**
    - Allow Drag & Drop or Button Click to move tasks from "Unsorted" to:
      - *Urgent (Red):* Due Today/Overdue. High pressure.
      - *Deadline (Yellow):* Buffer zone. Due soon.
      - *Admin (Blue):* Low energy, boring, maintenance.
      - *Creative (Purple):* High dopamine, fun, flow state.

3.  **Sprint Mode (Execution)**
    - User selects a Bucket based on *current mood*.
    - **UI Rule:** Show a "Time Elapsed" timer (Count Up), NOT a countdown timer (to reduce anxiety).
    - **Focus View:** Hide all other buckets. Show only tasks in the active bucket.

4.  **Guard Rails (Hard-Coded Rules)**
    - **Daily Highlight:** User selects ONE task as the "Win" for the day.
    - **4 PM Rule:** If time > 16:00 AND Daily Highlight is not done -> Trigger "Emergency Mode" alert.
    - **9 PM Hard Stop:** Visual indicator that the workday is over.

5.  **Kaizen Log (PDCA Cycle)**
    - Post-Sprint Trigger: When stopping a timer, ask:
      - "How did it feel?" (Mood Selector: üî• Flow, üòê Okay, üò´ Drained)
      - "What did you get done?" (Short text log)
    - Store this data for pattern recognition.

# Tech Stack
- React (Functional Components, Hooks)
- Tailwind CSS (Mobile-first, clean UI)
- LocalStorage (Data persistence)
- Lucide React (Iconography)

# Output
Generate the complete `App.jsx` file. Ensure the code handles state management for the timer, task transitions, and the specific time-based logic (4 PM/9 PM) automatically.
